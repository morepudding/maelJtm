Choix Cloud — Scores et verdict

Référence : slide 14

Critères pondérés : Légal & Souveraineté (20%) · Sécurité & Confiance (25%) · Indépendance technique (25%) · Maîtrise financière (20%) · Écologie / Green IT (10%)

#FournisseurScore /5ForcesFaiblesses
🥇 OVHcloud 4,50 100 % RGPD, champion Green IT, bande passante prévisible Moindre interopérabilité écosystème Microsoft
🥈 Scaleway 4,40 Souveraineté totale, facturation claire Briques avancées moins poussées
🥉 Microsoft Azure 4,10 Intégration AD / Office 365 / Power BI native Soumis au CLOUD Act, coûts moins prévisibles
4 Google Cloud 3,60 Puissance technique CLOUD Act + coûts imprévisibles
5 AWS 3,35 Catalogue de services le plus large CLOUD Act, nécessite spécialiste FinOps dédié
Verdict — OVHcloud retenu

L'intégration technologique ne fait pas tout : nous devons protéger légalement nos données (RGPD européen) et maîtriser notre budget avec certitude. Google Cloud et AWS ont été écartés car soumis au CLOUD Act — imprévisibilité financière et juridique pour une PME comme BricoLoc.

Annexe 05

Lexique des termes techniques

Glossaire complet pour les non-techniciens

ACID
Propriétés de fiabilité des transactions : Atomicité, Cohérence, Isolation, Durabilité
Active Directory (AD)
Annuaire Microsoft gérant les comptes et droits d'accès de l'entreprise
AMQP
Protocole de messagerie asynchrone utilisé par RabbitMQ
API
Contrat technique permettant à deux logiciels de communiquer
API Gateway
Point d'entrée unique sécurisant et routant les requêtes vers les modules
API REST
Style d'API sur HTTP utilisant des ressources et des verbes (GET, POST, PUT, DELETE)
B2B
Business-to-Business — ventes entre entreprises
B2C
Business-to-Consumer — ventes aux particuliers
Batch
Traitement automatique périodique (ici : le CSV quotidien SAP → BricoLoc)
CI/CD
Intégration Continue / Déploiement Continu — automatisation tests et déploiement
CLOUD Act
Loi US permettant l'accès aux données d'entreprises américaines, même en Europe
Conteneur / Docker
Empaquette une application avec ses dépendances pour un déploiement identique partout
CSV
Format texte (Comma-Separated Values) — export stocks depuis SAP
DMZ
Zone Démilitarisée — réseau exposé à Internet mais isolé du réseau interne
EJB
Enterprise JavaBeans — composants serveur de l'architecture actuelle (Java EE)
Entra ID
Nouveau nom d'Azure Active Directory, service d'identité cloud Microsoft
EOL
End of Life — fin de support officiel d'un logiciel (plus de correctifs de sécurité)
ERP
Logiciel de gestion intégré (ici : SAP Business One)
ESB
Enterprise Service Bus — middleware centralisant les communications SOA
Event-Driven
Architecture où les composants communiquent via événements asynchrones
FinOps
Discipline qui optimise et rend prévisibles les coûts du Cloud
FTP
Protocole de transfert de fichiers — « versioning » rudimentaire chez BricoLoc
Green IT
Réduction de l'empreinte écologique du numérique
IAM
Gestion centralisée des identités et des droits d'accès
IIS
Serveur web Microsoft hébergeant le service WCF VB.NET
Java EE
Java Enterprise Edition — plateforme d'entreprise (remplacée par Jakarta EE)
JPA
Java Persistence API — standard d'accès aux bases de données en Java
JWT
Jeton numérique signé pour authentifier un utilisateur entre requêtes
Kubernetes
Orchestrateur de conteneurs (non retenu — trop complexe pour 5 devs)
Maven
Outil de build et gestion de dépendances pour projets Java
Microservices
Services très fins, indépendants, chacun avec sa propre base de données
Monolithe modulaire
Un seul bloc déployable, organisé en modules isolés avec interfaces définies
Multi-tenant
Plusieurs clients partagent une même instance tout en isolant leurs données
N-tiers
Architecture en couches (présentation → logique → données) — modèle actuel
OpenAPI / Swagger
Standard de documentation interactive d'APIs REST
P2P
Peer-to-Peer — échange direct entre utilisateurs (location entre particuliers)
P95
95ème percentile — 95 % des requêtes traitées sous ce temps
PCI-DSS
Norme de sécurité des paiements par carte bancaire
PL/SQL
Langage procédural d'Oracle pour la logique en base de données
PL/pgSQL
Équivalent PostgreSQL du PL/SQL — facilite la migration depuis Oracle
RabbitMQ
Message broker open-source choisi pour la synchronisation temps réel des stocks
RBAC
Contrôle d'accès par rôles (CLIENT, PRO, LOGISTICIEN, ADMIN, PARTENAIRE)
Redis
Base en mémoire utilisée comme cache pour accélérer le catalogue
RGPD
Loi européenne sur la protection des données personnelles
RPO
Perte de données maximale acceptable en cas d'incident (cible : < 1h)
RTO
Délai maximal de remise en service après incident (cible : < 4h)
SaaS
Logiciel hébergé et accessible en ligne (modèle cible marque blanche)
SAP Business One
ERP utilisé par BricoLoc pour la logistique et les stocks
Scale-out
Ajout de serveurs pour absorber plus de charge
SLA
Engagement de niveau de service (cible : ≥ 99,5 % de disponibilité)
SOA
Architecture orientée services avec gouvernance centralisée
SPOF
Composant unique dont la panne bloque tout le système
Spring Boot
Framework Java open-source retenu pour BricoLoc 2.0
Strangler Fig
Migration progressive : le nouveau enveloppe l'ancien, module par module
TCO
Coût total de possession (licence + infra + formation + exploitation)
vRack
Réseau privé virtuel OVHcloud isolant les serveurs du réseau public
WAR / JAR
Formats de packaging Java déployables sur un serveur d'application
WCF
Framework Microsoft pour services web — composant BricoLoc dont le code est perdu
